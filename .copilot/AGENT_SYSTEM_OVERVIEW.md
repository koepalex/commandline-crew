# Commandline Crew - Agent System Overview

## 🚀 What Is Commandline Crew?

Commandline Crew is an AI-powered agent system for the terminal that provides specialized agents with different expertise areas. Each agent can be invoked independently or orchestrated together to handle complex development workflows.

---

## 🤖 Available Agents

### 1. @knowledgebase-wizard
**Purpose**: Knowledge discovery and documentation search

**Capabilities**:
- Search local knowledge bases (PDFs, markdown, text)
- Query official documentation online
- Find code examples and best practices
- Answer "How do I use X?" questions

**Usage**:
```bash
copilot --agent knowledgebase-wizard -p "Search mqtt for: clean session false"
copilot --agent knowledgebase-wizard -p "How do I use async/await?"
```

**Tools**: grep, glob, view, web_search, web_fetch

---

### 2. @quality-pal
**Purpose**: Code quality assurance and quality gate

**Capabilities**:
- Run linters and style checkers
- Validate .editorconfig compliance
- Analyze code for anti-patterns
- Detect outdated language/framework patterns
- Identify modern language features to use
- Execute builds and compile code
- Run test suites
- Generate detailed markdown quality reports
- Classify findings by severity (High/Medium/Low)

**Usage**:
```bash
copilot --agent quality-pal -p "Review src/api/handler.ts for quality"
copilot --agent quality-pal -p "Run full quality assurance on the codebase"
copilot --agent quality-pal -p "Run linters, build, and test suite"
```

**Tools**: grep, glob, view, powershell, task

**Quality Gate Role**:
Acts as a quality gate for code generated by other agents. Quality-pal reports findings, classifies severity, and prevents low-quality code from being merged.

---

### 3. @orchestrator
**Purpose**: Multi-agent workflow coordination

**Capabilities**:
- Decompose complex tasks into agent-specific subtasks
- Delegate work to specialized agents
- Combine results from multiple agents
- Manage workflow state and transitions

**Status**: Framework ready, ready for integration

**Tools**: Coordinates other agents

### Query Syntax
```bash
# Search specific KB
copilot --agent knowledgebase-wizard -p "Search mqtt for: your question"

# Search multiple KBs
copilot --agent knowledgebase-wizard -p "Search mqtt and docs for: question"

# Search all registered KBs (default)
copilot --agent knowledgebase-wizard -p "How do I use X?"
```

## 🏗️ Quality Pal Severity Classification

Every finding from quality-pal is classified:

### 🔴 HIGH
- **What**: Critical issues blocking merge
- **Examples**: Compilation errors, test failures, security vulnerabilities, breaking changes
- **Action**: Must fix before merge
- **Criteria**: Affects functionality, security, or stability

### 🟡 MEDIUM
- **What**: Important improvements
- **Examples**: Deprecated APIs, non-idiomatic code, performance issues
- **Action**: Should fix if possible
- **Criteria**: Affects quality, maintainability, or future compatibility

### 🟢 LOW
- **What**: Nice-to-have improvements
- **Examples**: Style issues, naming conventions, documentation gaps
- **Action**: Consider in next iteration
- **Criteria**: Doesn't affect functionality

## 🎯 Quality Gate Workflow

When quality-pal is used as a quality gate:

```
1. Other Agent Generates Code
   └─ refactor-agent, test-specialist, etc.

2. Quality-Pal Reviews
   ├─ Runs linters
   ├─ Validates style
   ├─ Checks patterns
   ├─ Builds project
   └─ Runs tests

3. Findings Classified
   ├─ 🔴 HIGH: Blocks merge
   ├─ 🟡 MEDIUM: Should fix
   └─ 🟢 LOW: Nice-to-have

4. Report Generated
   └─ Markdown with suggestions

5. Decision Made
   ├─ PASS: Code ready to merge
   ├─ REVISE: Address issues and retry
   └─ MERGE WITH NOTES: Document deferred issues
```

## 🚀 Getting Started

```bash

mkdir ./resources/my-docs
cp your-doc.pdf ./resources/my-docs/

# Register in .copilot/knowledge-bases.md
# Add row: | my-docs | Description | `./resources/my-docs` | pdf |

copilot --agent knowledgebase-wizard -p "Search my-docs for: your question"
```

```bash
# Review specific file
copilot --agent quality-pal -p "Review src/handler.ts for quality"

# Full codebase check
copilot --agent quality-pal -p "Run linters, build, and tests"
```

### 3. Workflow Coordination
```bash
# Have orchestrator delegate to multiple agents
/agent
# (then select orchestrator from the list)
```

## ❓ FAQ

**Q: Can I use multiple agents in one session?**
A: Yes! Use the orchestrator or manually invoke agents in sequence.

**Q: What if an agent doesn't find what I'm looking for?**
A: KB-Wizard supplements local search with web search. You can also add more KB sources.

**Q: Can agents modify code?**
A: Quality-pal reports findings but doesn't modify. Other agents (when created) will handle implementation.

**Q: How do I add more PDFs to a KB?**
A: Copy files to the KB folder and quality-pal/KB-wizard will find them automatically.

**Q: Is quality-pal mandatory before merge?**
A: It acts as a quality gate. Use it to ensure code meets standards before merge.
