## What Quality Pal Does

### Core Responsibilities
1. **Code Review** - Analyzes code for quality issues
2. **Linting** - Runs language-specific linters
3. **Style Validation** - Checks `.editorconfig` compliance
4. **Pattern Analysis** - Detects anti-patterns and outdated practices
5. **Modern Features** - Identifies opportunities to use latest language features
6. **Build Validation** - Runs compilers and build systems
7. **Test Execution** - Runs test suites and reports results
8. **Report Generation** - Creates detailed markdown quality reports

### Severity Classification

Every finding is classified into one of three levels:

**рЯФі HIGH** - Critical issues that must be fixed
- Compilation errors, test failures
- Security vulnerabilities
- Breaking API changes
- Unhandled exceptions
- **Action**: Must fix before merge

**рЯЯ° MEDIUM** - Important improvements
- Deprecated API usage
- Non-idiomatic code
- Performance anti-patterns
- **Action**: Should fix if possible

**рЯЯҐ LOW** - Nice-to-have improvements
- Style inconsistencies
- Naming conventions
- Documentation gaps
- **Action**: Consider in next iteration

## FAQ

**Q: Can quality-pal fix the code?**
A: No, it only reports findings and suggests fixes. Other agents handle implementation.

**Q: What if the build fails?**
A: Quality-pal reports the error, suggests fixes, but doesn't resolve it.

**Q: How are findings classified?**
A: By impact: HIGH (blocks merge), MEDIUM (should fix), LOW (nice-to-have).

**Q: Can it check multiple languages?**
A: Yes, it auto-detects language from file extensions and imports.

**Q: What about custom linters?**
A: Quality-pal uses standard tools; custom rules go in config files.
